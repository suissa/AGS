# 00 — AGS Project Architecture & Repository Bootstrap

Crie a base arquitetural do **AGS — Agentic GitHub System**.

## Objetivo

Criar estrutura inicial do projeto, CLI base, configuração, documentação e testes mínimos.

## Contexto

AGS é um runtime GitHub-native para desenvolvimento com Agentes de IA. Ele automatiza GitHub Projects, Issues, PRs, Actions, Wiki, CTID, Cognitive Provenance, Project Sync Rules e execução agentica.

## Estrutura esperada

```txt
ags/
  package.json
  tsconfig.json
  README.md
  .env.example
  src/
    cli/
    core/
    github/
    projects/
    ctid/
    events/
    llm/
    sync/
    tui/
    config/
    utils/
  tests/
  docs/
```

## Entregáveis

- `package.json`
- `tsconfig.json`
- `.env.example`
- `src/cli/index.ts`
- `src/core/ags-core.ts`
- `src/config/load-config.ts`
- `README.md`
- testes iniciais

## Validação

```bash
npm install
npm run build
npm test
```

## Critérios de aceite

- O projeto compila.
- O CLI inicia sem erro.
- Existe separação clara de módulos.
- O README explica a proposta do AGS.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
