# 01 — CLI Commands: `ags project create` e `ags project sync`

Implemente os comandos oficiais do AGS.

## Objetivo

Criar:

```bash
ags project create
ags project sync
```

Não crie `ags init` nem `ags repo create`. No AGS, o Project é a unidade operacional raiz. Repo, Projects, Wiki, Workflows, CTID e Actions são derivados do projeto.

## `ags project create`

Deve perguntar:

- nome do projeto;
- owner/repo;
- se deve criar repo remoto;
- modo: `simple` ou `allascode`;
- se `TASKS_GENERATED=true`;
- se `TASKS_GENERATED=true`, path relativo do `.md`;
- se deve ativar TUI.

Deve gerar `.ags/config.json`.

## `ags project sync`

Deve carregar `.ags/config.json`, iniciar event bus, Project Sync Rules Engine e TUI se habilitada.

## Critérios de aceite

- CLI interativo e também por flags.
- Config persistida.
- `project sync` falha com mensagem clara se `.ags/config.json` não existir.
