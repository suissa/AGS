# 03 — GitHub GraphQL Project V2 Provisioning

Implemente a camada GitHub GraphQL para Projects v2.

## Objetivo

Criar Projects v2 automaticamente para os modos `simple` e `allascode`.

## Modo simple

Criar apenas:

```txt
AGS - Execution
```

Campos mínimos:

- Status
- Priority
- CTID
- Task Type

## Modo allascode

Criar:

```txt
AGS - Backlog
AGS - Execution
AGS - Bug Tracker
AGS - Cognitive
AGS - Security
```

Campos comuns:

- CTID
- Status
- Priority
- Task Type
- Agent
- Last Sync
- Source

Status sugeridos: `Backlog`, `Ready`, `In Progress`, `In Review`, `Blocked`, `Done`, `Failed`.

## Arquivos esperados

```txt
src/projects/github-graphql-client.ts
src/projects/project-provisioner.ts
src/projects/project-mode.ts
src/projects/project-fields.ts
src/projects/project-views.ts
tests/project-provisioner.test.ts
```

## Critérios de aceite

- `simple` cria 1 project.
- `allascode` cria 5 projects.
- Provisioner idempotente.
- Logs claros para TUI.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
