# 04 — CTID: Cognitive Trace ID System

Implemente o sistema CTID.

## Objetivo

Criar identificador global de correlação para cada tarefa agentica.

## Definição

CTID não substitui issue number. Ele correlaciona issue, branch, commits, PR, Project item, LLM reports, traces, contexto final e Wiki snapshots.

## Formato sugerido

```txt
ctid_{repo_slug}_{issue_number}_{short_hash}
```

Exemplo:

```txt
ctid_auth_service_42_a91f33c9
```

## Arquivos esperados

```txt
src/ctid/ctid.ts
src/ctid/ctid-registry.ts
src/ctid/ctid-store.ts
src/ctid/ctid-types.ts
tests/ctid.test.ts
```

## Registry local

Criar `.ags/ctid-registry.json`.

## Critérios de aceite

- Gera CTID.
- Registra CTID.
- Lê CTID existente.
- Atualiza status.
- Testa colisão básica.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
