# 06 — Internal Event Bus

Implemente o event bus interno do AGS.

## Objetivo

Criar camada local de eventos para sincronização, logs estruturados, replay futuro, TUI e Project Sync Rules.

## Regras

- Primeira versão com `EventEmitter`.
- Interface preparada para NATS/Kafka no futuro.
- Todo evento ligado a tarefa deve possuir CTID.
- Todo evento deve possuir timestamp.
- Todo evento deve ser serializável em JSON.

## Arquivos esperados

```txt
src/events/event-bus.ts
src/events/event-types.ts
src/events/event-store.ts
src/events/event-log-writer.ts
tests/event-bus.test.ts
```

## Evento mínimo

```ts
type AGSEvent = {
  id: string;
  type: string;
  ctid?: string;
  source: string;
  payload: Record<string, unknown>;
  createdAt: string;
}
```

## Persistência

Salvar eventos em `.ags/events.jsonl`.

## Critérios de aceite

- Publica evento.
- Consome evento.
- Salva em JSONL.
- TUI escuta eventos.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
