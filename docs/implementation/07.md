# 07 — LLM Plugin Layer

Implemente a camada plugável de LLMs.

## Objetivo

Permitir configurar diferentes agentes usando diferentes provedores e modelos.

## Provedores previstos

- OpenAI-compatible API
- Anthropic-compatible API
- local HTTP endpoint
- mock provider para testes

## Funções obrigatórias

- gerar breakdown de tarefa;
- validar formato do TASKS.md;
- revisar código;
- gerar Cognitive Provenance Report;
- consolidar outputs no Loop-Until-Done.

## Arquivos esperados

```txt
src/llm/llm-provider.ts
src/llm/llm-plugin-layer.ts
src/llm/providers/openai-compatible.ts
src/llm/providers/anthropic-compatible.ts
src/llm/providers/local-http.ts
src/llm/providers/mock-provider.ts
tests/llm-plugin-layer.test.ts
```

## Interface esperada

```ts
interface LLMProvider {
  name: string;
  model: string;
  complete(input: LLMRequest): Promise<LLMResponse>;
}
```

## Critérios de aceite

- Suporta mock provider.
- Valida `TASKS.md`.
- Gera breakdown em JSON.
- Emite evento `llm.used`.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
