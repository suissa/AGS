# 08 — TASKS_GENERATED Validator

Implemente o validador de arquivos de tarefas já gerados.

## Objetivo

Quando `TASKS_GENERATED=true`, o CLI deve pedir o path relativo do `.md` das tasks e validar o formato para o padrão AGS.

## Regras

- Não executar tarefas.
- Não alterar código do projeto.
- Apenas validar estrutura.
- Se inválido, gerar relatório de correção.
- Se válido, registrar no `.ags/config.json`.

## Formato mínimo esperado

```md
# Tasks

> AGS directive...

- [ ] 🚀 #ID - Nome da tarefa
  - [ ] Subtarefa
```

Ou:

```md
- [ ] 🚀 TASK-001 - Nome da tarefa
```

## Arquivos esperados

```txt
src/tasks/tasks-md-validator.ts
src/tasks/tasks-md-parser.ts
src/tasks/tasks-ai-validator.ts
src/tasks/tasks-types.ts
tests/tasks-md-validator.test.ts
```

## Critérios de aceite

- Valida arquivo existente.
- Detecta checkbox ausente.
- Detecta task sem ID.
- Detecta subtarefa mal indentada.
- Gera relatório claro.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
