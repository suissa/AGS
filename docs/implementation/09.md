# 09 — TUI em tempo real estilo git + kubectl + htop

Implemente a TUI do AGS.

## Objetivo

Criar interface terminal em tempo real mostrando modo, projects, eventos, CTIDs, tarefas, agentes, CI status, LLM usage, erros e bloqueios.

## Estilo

Mistura de:

- `git status`
- `kubectl get pods -w`
- `htop`
- dashboard de CI/CD

## Layout mínimo

```txt
┌ AGS Runtime ────────────────────────────────┐
│ Mode: allascode | Repo: owner/repo          │
├ Projects ──────────────────────────────────┤
│ Backlog: 12 | Execution: 3 | Bug: 1         │
├ Events ────────────────────────────────────┤
│ task.created ...                            │
│ llm.used ...                                │
├ Agents ────────────────────────────────────┤
│ builder-a running                           │
│ critic-b idle                               │
├ CI/CD ─────────────────────────────────────┤
│ lint ok | test running | build pending      │
└─────────────────────────────────────────────┘
```

## Arquivos esperados

```txt
src/tui/ags-tui.ts
src/tui/tui-state.ts
src/tui/tui-renderer.ts
src/tui/tui-events-adapter.ts
tests/tui-state.test.ts
```

## Critérios de aceite

- Recebe eventos do Event Bus.
- Atualiza tela em tempo real.
- Funciona em fallback sem TTY.
- Não quebra em CI.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
