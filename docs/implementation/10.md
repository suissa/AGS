# 10 — Cognitive Provenance Report

Implemente o relatório cognitivo do AGS.

## Objetivo

Gerar relatório por tarefa contendo CTID, task id, issue, branch, agentes, modelos, artefatos, testes, decisões, riscos, custo estimado, tempo total e contexto final.

## Regras

- Não persistir chain-of-thought bruto.
- Persistir semantic traces, decisões resumidas e evidências.
- Gerar Markdown e JSON.
- Relatório deve poder ser comentado na issue/PR.
- Relatório deve alimentar Wiki.

## Arquivos esperados

```txt
src/provenance/provenance-report.ts
src/provenance/provenance-schema.ts
src/provenance/provenance-writer.ts
src/provenance/provenance-markdown.ts
tests/provenance-report.test.ts
```

## Saída

```txt
.ags/provenance/{ctid}.json
docs/lifecycle/provenance/{ctid}.md
```

## Critérios de aceite

- Gera JSON válido.
- Gera Markdown legível.
- Inclui CTID.
- Pode ser anexado à issue.
- Não contém CoT bruto.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
