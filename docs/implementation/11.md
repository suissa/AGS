# 11 — Loop-Until-Done Convergence Engine

Implemente o Loop-Until-Done.

## Objetivo

Executar ciclos iterativos até a tarefa convergir.

## Pronto significa

- testes obrigatórios passam;
- reviewers não encontram inconsistência bloqueante;
- não há duplicação relevante;
- artefatos finais foram consolidados;
- Cognitive Provenance Report foi gerado.

## Fluxo

```txt
run builders
collect outputs
cross review
run tests
detect conflicts
unify
validate again
if ok -> converged
else -> next loop
```

## Arquivos esperados

```txt
src/convergence/loop-until-done.ts
src/convergence/convergence-state.ts
src/convergence/convergence-checker.ts
src/convergence/context-artifact-writer.ts
tests/loop-until-done.test.ts
```

## Arquivo final obrigatório

Gerar:

```txt
.ags/contexts/{task_id}-context-{Date.now}.txt
```

## Critérios de aceite

- Respeita max iterations.
- Gera contexto final.
- Emite `task.converged`.
- Emite `task.failed` se exceder limite.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
