# 13 — GitHub Wiki Snapshot Generator

Implemente o gerador de Wiki do AGS.

## Objetivo

Criar e atualizar páginas de Wiki automaticamente com changelog, versões antigas, snapshots de arquitetura, CTIDs concluídos, relatórios cognitivos resumidos e diferenças entre versões.

## Estrutura sugerida

```txt
Home.md
Changelog.md
Versions/
  v0.1.0.md
Architecture/
  Overview.md
Cognitive/
  CTID-Index.md
```

## Arquivos esperados

```txt
src/wiki/wiki-generator.ts
src/wiki/wiki-git-client.ts
src/wiki/changelog-generator.ts
src/wiki/version-snapshot.ts
tests/wiki-generator.test.ts
```

## Regras

- Wiki opcional.
- Deve operar em dry-run.
- Se wiki não existir, informar instrução clara.
- Deve gerar Markdown local mesmo sem push.

## Critérios de aceite

- Gera Home.
- Gera Changelog.
- Gera página por versão.
- Gera índice de CTIDs.
- Funciona em dry-run.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
