# 14 — CI/CD Mapping Engine

Implemente o mapeamento entre CI/CD e Project Sync.

## Objetivo

Conectar resultados de GitHub Actions ao estado dos Projects.

## Pipeline previsto

Suportar configuração para lint, test, build, sonar, e2e, docker, deploy, sentry, snyk, dependabot e KiloCode/Copilot review quando existir.

## Mapeamento

```txt
lint failed -> Blocked
test failed -> Bug Tracker
build failed -> Blocked
sonar failed -> Quality Issue
e2e failed -> Bug Tracker
docker failed -> Deployment Blocked
deploy success -> Done/Released
dependabot alert -> Security
```

## Arquivos esperados

```txt
src/cicd/cicd-mapping-engine.ts
src/cicd/workflow-run-parser.ts
src/cicd/default-cicd-rules.ts
src/cicd/cicd-types.ts
tests/cicd-mapping-engine.test.ts
```

## Critérios de aceite

- Lê workflow runs.
- Converte status em eventos AGS.
- Atualiza Project Sync Rules Engine.
- Funciona sem deploy configurado.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
