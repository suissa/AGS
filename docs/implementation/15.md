# 15 — Final Integration: AGS End-to-End Runtime

Integre todos os módulos do AGS.

## Objetivo

Criar fluxo completo:

```bash
ags project create
ags project sync
```

## `ags project create` deve

- carregar perguntas interativas;
- criar config;
- criar repo se solicitado;
- criar Projects conforme modo;
- instalar workflows/templates;
- configurar CTID;
- validar TASKS.md se `TASKS_GENERATED=true`;
- iniciar TUI opcional.

## `ags project sync` deve

- iniciar event bus;
- carregar CTID registry;
- ler GitHub state;
- aplicar Project Sync Rules;
- processar CI/CD mappings;
- gerar provenance reports;
- atualizar Wiki se habilitada;
- manter TUI atualizada.

## Modos

### simple

- 1 Project: `AGS - Execution`
- CTID básico
- Issues/PR/Actions sync
- 1 builder + 1 reviewer/tester opcional

### allascode

- Backlog
- Execution
- Bug Tracker
- Cognitive
- Security
- CTID registry
- Event bus
- Wiki snapshots
- Multi-agent execution
- Loop-Until-Done
- Cognitive Provenance
- CI/CD mapping
- TUI completa

## Validação final

```bash
npm run build
npm test
ags project create --dry-run
ags project sync --dry-run
```

## Critérios de aceite

- Simple mode funciona em dry-run.
- AllasCode mode gera estrutura completa em dry-run.
- Nenhum token necessário para dry-run.
- Com token válido, GitHub API real funciona.
- README explica cada etapa e por que é usada.


## Regras gerais para a IA implementadora

- Implemente apenas o escopo deste arquivo.
- Crie ou atualize testes automatizados.
- Atualize documentação quando necessário.
- Não persista chain-of-thought bruto; use semantic traces estruturados.
- Não avance para a próxima etapa sem validação local.
- Use TypeScript/Node.js salvo se o projeto já tiver outra stack explicitamente definida.
- Nunca hardcode tokens, API keys ou secrets.
- Todo módulo deve funcionar em dry-run quando depender de GitHub real.
